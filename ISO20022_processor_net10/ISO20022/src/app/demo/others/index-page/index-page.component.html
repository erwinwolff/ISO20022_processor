ISO 20022 Financial Messaging Standard


ISO 20022 is a global financial messaging standard created by the International Organization for Standardization (ISO) to modernize how banks, payment systems, and financial institutions exchange data.

Simple definition:

It is a universal language for payments and financial transactions — replacing older banking formats with richer, structured data.



Why It Exists

Before ISO 20022, different payment networks used incompatible message formats.

  • Limited transaction details
  • Manual reconciliation
  • Payment delays
  • Compliance difficulties (AML, sanctions screening)

ISO 20022 solves this by using structured XML data so every system understands the same meaning.



Core Characteristics

Feature Description Benefit
Structured Data XML tagged fields Automatic processing
Rich Information Sender, receiver, invoice, purpose Better tracking
Global Interoperability Common international format Cross-border compatibility
Extensibility Can evolve without breaking systems Future-proof payments


Example Payment Message (Simplified)

<CstmrCdtTrfInitn>
  <PmtInf>
    <Dbtr>Alice Corp</Dbtr>
    <Cdtr>Bob Ltd</Cdtr>
    <Amt Ccy="EUR">1250.00</Amt>
    <RmtInf>Invoice 45821</RmtInf>
  </PmtInf>
</CstmrCdtTrfInitn>

Each field has a defined meaning — no ambiguity.



Where ISO 20022 Is Used

  • Domestic payments (instant & batch)
  • Cross-border payments
  • Card transactions
  • Securities trading & settlement
  • Central bank reporting
